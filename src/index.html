UI-Elements

Headings

For headlines are h1, h2 and h3 available. All use Avenir Black. While h1 is displayed in $darkblue, h2 and h3 use $lightblue. Headlines are always displayed centered.
h1 is used for the most important headlines. Subsections are titled with h2. To reduce complexity h3 should only be used if absolutely necessary.

heading 1

heading 2

heading 3

<h1> heading 1 </h1>
<h2> heading 2 </h2>
<h3> heading 3 </h3>

Paragraph

Running text is written in Avenir Light and left-aligned justified. The text color is either $black or $darkgray and always displayed on white background.
It is used for all longer text passages.

CSS Methodology

Preprocessor

CSS on its own can be fun, but stylesheets are getting larger, more complex, and harder to maintain. This is where a preprocessor can help. Sass lets you use features that don't exist in CSS yet like variables, nesting, mixins, inheritance and other nifty goodies that make writing CSS fun again.

Naming Conventions

The naming conventions for the single sass-files prescribes that every partial has to be names after the main UI-element it handles. To seperate partials from main files the partials have to start with an underscore.

All classes follow the spinal-case convention. This means that if a class name contains more than one word it they have to be seperated with a hyphen.

Code Style Rules

  • Indentation: 2 spaces
  • Sass-File: write element before class
  • Curly Braces: start directly after selectors
  • Semicolon: has to be used in all style and script files
  • Trailing Whitespaces: have to be avoided

File Structure

Every UI-Element is defined in its own sass-file for a more clear arrangement. Every sass-file should be imported in the 'uielements.scss' file. Thereby the developers are able to choose the desired UI-Elements easily. Global and custom style rules can be defined in the main sass file. The sass files get combined to one main.css file in the 'dist' directory which has to be included in the html file.

The 'src' directory should contain the following subdirectories: 'assets', 'scripts' and 'styles'. Images and fonts have to be saved in the assets directory. The scripts directory holds any JavaScript files. Any style related files are stored in the styles directory.
